Git 2.23+ チートシート

初学者が「初回 push 済み」後の日常作業でよく使う操作

前提：Git 2.23以降（git switch / git restore が使えるバージョン）

💡 コマンドをクリックするとクリップボードにコピーできます

🔍 1. 変更を確認する

目的 コマンド ポイント
追跡状況を見る git status まずはこれを使う
作業ツリー ↔ HEAD の差分 git diff コミット前に確認する
ステージ済み ↔ HEAD の差分 git diff --staged --cached と同じ意味

✅ 2. 変更を追加・確定する

目的 コマンド ポイント
ステージへ追加 git add <path> / git add . 変更をコミット準備に入れる
コミット作成 git commit -m "メッセージ" 変更を履歴に保存する
前回コミットを上書き git commit --amend 注意: push 前なら安全に修正できる

🌿 3. ブランチを扱う

目的 コマンド ポイント
ブランチ作成 + そのまま移動 git switch -c feature/foo -c は create の略
既存ブランチへ移動 git switch develop 旧 checkout よりシンプル
最新リモート追跡ブランチを元に作成 git switch -c hotfix/bug origin/main 最新の origin/main を基点に作成
ブランチ名を変更 git branch -m old-name new-name ローカルブランチ名をリネームする
マージ（早送り優先） git merge --ff-only feature/foo コンフリクト時に止めることができる
Rebase（履歴を直線に） git rebase origin/main 注意: 履歴をきれいに保つことができる
ブランチを削除 git branch -d feature/foo -D で強制削除できる

🔄 4. リモートと同期する

目的 コマンド ポイント
最新を取得＋自動マージ (FF 限定) git pull --ff-only 履歴を汚さずに更新できる
取得だけ git fetch 後で手動マージまたは rebase する
変更を共有 git push 初回は -u origin <branch> を付ける
タグを共有 git push --tags タグもリモートに送信する

↩️ 5. 変更を取り消す／やり直す

目的 コマンド ポイント
作業ツリーを最後のコミットに戻す git restore <file> 旧 checkout -- よりシンプル
ステージから外す git restore --staged <file> 旧 reset より明確
共有履歴を壊さず打ち消す git revert <hash> 打ち消しコミットを生成する

📦 6. 作業を一時退避する

目的 コマンド ポイント
退避 git stash push -m "wip: form layout" stash save は非推奨
取り出して削除 git stash pop 最新の stash を適用して削除する
取り出しのみ git stash apply <id> stash を削除せずに適用する
一覧 git stash list 保存した stash を確認する

🔎 7. 履歴を調べる & クリーンアップ

目的 コマンド ポイント
履歴をグラフ表示 git log --oneline --graph --decorate --all 全体を俯瞰して確認できる
すべての操作履歴 git reflog ミスを復元する最後の手段
未追跡ファイルを一掃 git clean -fd 破壊的。実行前に -n で確認

どのバージョンから使える？

  • git switch / git restore → Git 2.23（2019年）以降
  • git stash push → Git 2.16（2018年）以降、stash save は非推奨

✔ これだけ覚えれば

  1. 確認 git status git diff
  2. 追加 git add .
  3. 確定 git commit -m "メッセージ"
  4. 共有 git push
  5. 同期 git pull --ff-only
  6. 枝分かれ git switch -c feature/new git merge feature/new git rebase origin/main
  7. 巻き戻し git restore <file> git revert <hash> git stash push

旧コマンドはドキュメントや Stack Overflow を読む "受け身の知識" として頭の片隅に置き、 自分で打つときは新コマンドだけ使う──これが混乱を避ける最短ルートです。
